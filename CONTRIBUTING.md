# Contributing

Contributions are welcome and will be fully credited.

## Etiquette

Be considerate and respectful toward maintainers and other contributors.

## Before Submitting

- Replicate the issue to ensure it's reproducible
- Check existing issues and PRs to avoid duplicates
- Verify your feature doesn't already exist in the codebase

## Requirements

- **Add tests** - Your patch won't be accepted without tests
- **Document changes** - Update README.md and relevant docs
- **Follow SemVer** - Don't break public APIs
- **One PR per feature** - Submit multiple PRs for multiple features
- **Clean history** - Squash intermediate commits before submitting

Happy coding!
